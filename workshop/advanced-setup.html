Module 2: Environment Setup

Setting up GitHub Codespaces for the multi-agent orchestrator

‚è±Ô∏è Duration: 10 minutes

üöÄ Launch GitHub Codespace

We'll use GitHub Codespaces to create a consistent development environment with all dependencies pre-configured.

  • Open the Repository
    Navigate to: https://github.com/khanasif1/demo-microsoft-agent-framework
  • Create Codespace
    Click the green "Code" button ‚Üí "Codespaces" tab ‚Üí "Create codespace on main"
  • Wait for Setup
    Codespace will automatically install Python dependencies. This takes ~2-3 minutes.
    • ‚ÑπÔ∏è What's being installed: Python 3.11+, FastAPI, aiohttp, BeautifulSoup4, lxml, Microsoft Agent Framework, and Azure OpenAI SDK.

    üìÇ Navigate to Advanced Project

    The repository contains multiple examples. We need to navigate to the concurrent orchestrator solution.

    Terminal Command

    cd _src/useKey/agentConcurrentOrchestrator

    Verify you're in the right directory by listing the contents:

    Terminal Command

    ls -la

    You should see:

    agentApp.py           # Main FastAPI application
requirements.txt      # Python dependencies  
.env.example         # Environment template
tools/               # Agent tools directory
static/              # Web UI files

    üì¶ Install Dependencies

    Install the project-specific dependencies for the concurrent orchestrator:

    Terminal Command

    pip install -r requirements.txt

    This installs additional packages needed for the multi-agent system:

    • fastapi - Modern web framework for building APIs
    • uvicorn - ASGI server for running FastAPI
    • aiohttp - Async HTTP client for web scraping
    • beautifulsoup4 & lxml - HTML parsing for news extraction
    • python-multipart - For handling form data
    ‚úÖ Expected output: You should see successful installation messages for each package.

    üîê Configure Azure OpenAI

    Set up your Azure OpenAI credentials for the agent orchestrator.

    Step 1: Copy Environment Template

    Terminal Command

    cp .env.example .env

    Step 2: Edit Environment File

    Open the .env file and add your Azure OpenAI credentials:

    Terminal Command

    code .env

    Update the file with your Azure OpenAI details:

    # Azure OpenAI Configuration
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_API_KEY=your-api-key-here
AZURE_OPENAI_API_VERSION=2024-02-15-preview
AZURE_OPENAI_DEPLOYMENT_NAME=gpt-4o
    ‚ö†Ô∏è Important: Make sure you have a GPT-4 or GPT-4o deployment in your Azure OpenAI resource. GPT-3.5 may not perform well for complex multi-agent coordination.
