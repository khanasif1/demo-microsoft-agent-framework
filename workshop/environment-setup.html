Module 2: Python Environment Setup

Create an isolated Python environment and install dependencies

‚è±Ô∏è Duration: 10 minutes

üéØ What You'll Accomplish

In this module, you'll create a Python virtual environment and install all the necessary packages for the Microsoft Agent Framework. By the end, you'll have everything ready to run the agent code.

üí° Why Virtual Environments? A virtual environment creates an isolated space for your project's dependencies. This prevents conflicts with other Python projects and keeps your global Python installation clean.

üìÅ Step 1: Navigate to the Project Folder

First, let's navigate to the agentquickstart folder where our code is located.

Open Your Terminal

  • Windows: Press Win + R, type cmd or powershell, and press Enter
  • macOS: Press Cmd + Space, type terminal, and press Enter
  • Linux: Press Ctrl + Alt + T

Navigate to the Folder

Use the cd command to change directory:

Windows (PowerShell/CMD)
cd C:\path\to\demo-microsoft-agent-framework\_src\agentquickstart
macOS/Linux
cd /path/to/demo-microsoft-agent-framework/_src/agentquickstart
‚ö†Ô∏è Replace the path: Make sure to replace /path/to/ with the actual path where you cloned or downloaded the repository.

‚úÖ Verify You're in the Right Place

List the files in the current directory:

Windows (PowerShell)
dir
macOS/Linux
ls

You should see these files:

azure_ai_basic.py
README.md
.env (might not be visible yet)
‚úÖ Great! If you see azure_ai_basic.py, you're in the correct folder.

üêç Step 2: Create a Python Virtual Environment

Now let's create a virtual environment. This creates a folder called .venv that will contain all our Python packages isolated from your system Python.

Create the Virtual Environment

Windows
python -m venv .venv
macOS/Linux
python3 -m venv .venv
üí° What's happening?
  • python -m venv - Runs Python's built-in venv module
  • .venv - The name of the folder to create (you can use any name, but .venv is conventional)

This command will take a few seconds to complete. You won't see much output, but a .venv folder will be created.

‚úÖ Verify Virtual Environment Creation

Check if the .venv folder was created:

Windows (PowerShell)
dir .venv
macOS/Linux
ls .venv

You should see folders like Scripts (Windows) or bin (macOS/Linux), Lib, and Include.

‚ö° Step 3: Activate the Virtual Environment

Before we can use the virtual environment, we need to activate it. This tells your terminal to use the Python interpreter and packages from the virtual environment instead of your system Python.

Activation Commands

Windows (PowerShell)
.venv\Scripts\Activate.ps1
‚ùå PowerShell Execution Policy Error? If you get an error about execution policy, run: 
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Then try activating again.
Windows (CMD)
.venv\Scripts\activate.bat
macOS/Linux
source .venv/bin/activate

‚úÖ Verify Activation

After activation, you should see (.venv) at the beginning of your command prompt: 

(.venv) C:\Users\YourName\agentquickstart>
‚úÖ Success! If you see (.venv) prefix, your virtual environment is active!
üí° Remember: You need to activate the virtual environment every time you open a new terminal window to work on this project.

üì¶ Step 4: Upgrade pip

pip is Python's package installer. Let's make sure we have the latest version before installing our dependencies.

Command (all platforms)
python -m pip install --upgrade pip

You'll see output showing pip being upgraded. This usually takes 10-30 seconds.

‚úÖ Done! Your pip is now up to date.

üìö Step 5: Install Required Packages

Now comes the main step - installing all the packages needed for the Microsoft Agent Framework. We'll install them from the requirements.txt file located in the parent directory.

What Packages Will Be Installed?

Package Purpose
agent-framework Microsoft Agent Framework core library
azure-identity Azure authentication and credential management
python-dotenv Load environment variables from .env files
pydantic Data validation and type hints
aiohttp Async HTTP client/server

Install All Packages

Run this command to install all dependencies at once:

Command
pip install -r ../requirements.txt
üí° Note: The ../requirements.txt path goes up one directory to the _src folder where the requirements file is located.

This installation will take 2-5 minutes depending on your internet speed. You'll see a lot of output as pip downloads and installs each package and its dependencies.

‚úÖ Verify Installation

After installation completes, verify the key packages are installed:

Command
pip list

Look for these packages in the output:

agent-framework        x.x.x
azure-identity         x.x.x
python-dotenv          x.x.x
pydantic               x.x.x
aiohttp                x.x.x
‚úÖ Perfect! If you see these packages listed, the installation was successful.

‚öôÔ∏è Step 6: Create Environment Configuration File

The agent needs to know where to connect to Azure AI services. We store this information in a .env file (environment file).

Create the .env File

Create a new file named .env in the agentquickstart folder:

Windows (PowerShell)
New-Item .env
macOS/Linux
touch .env

Add Configuration Placeholders

Open the .env file in your text editor (VS Code, Notepad, etc.) and add:

.env file content
# Azure AI Project Configuration
AZURE_AI_PROJECT_ENDPOINT=your-endpoint-here
AZURE_AI_MODEL_DEPLOYMENT_NAME=your-deployment-name-here
‚ö†Ô∏è Don't worry about the values yet! We'll fill in the actual values in the next module when we set up Azure AI. For now, just create the file with these placeholders.
üí° Important: The .env file should never be committed to version control (Git) because it contains sensitive information. It should already be in .gitignore.

‚úÖ Environment Setup Checklist

Before moving to the next module, verify you've completed all these steps:

  • Navigated to the agentquickstart folder
  • Created virtual environment with python -m venv .venv
  • Activated virtual environment (seeing (.venv) prefix)
  • Upgraded pip to the latest version
  • Installed all packages from requirements.txt
  • Created .env file with placeholder values
üéâ Excellent work! Your Python environment is now fully configured. In the next module, we'll set up Azure AI and get the values needed for the .env file.

üìñ Quick Reference

Activating Virtual Environment (Future Sessions)

Every time you start a new terminal session, you'll need to:

  1. Navigate to the project folder: cd /path/to/agentquickstart
  2. Activate the virtual environment:
    • Windows (PowerShell): .venv\Scripts\Activate.ps1
    • Windows (CMD): .venv\Scripts\activate.bat
    • macOS/Linux: source .venv/bin/activate

Deactivating Virtual Environment

When you're done working, you can deactivate the virtual environment:

deactivate
