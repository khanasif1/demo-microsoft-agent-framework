Module 1: GitHub Codespaces Setup

Launch your cloud development environment in seconds

⏱️ Duration: 5 minutes

What You'll Accomplish

In this module, you'll launch GitHub Codespaces - a fully configured cloud development environment that runs in your browser. No installation, no configuration, just instant coding!

What's Pre-Installed: Python 3.11+, all dependencies, VS Code, Git, and all required packages!

Step 1: Navigate to the Repository

  1. Open the Repository

    Go to https://github.com/khanasif1/demo-microsoft-agent-framework

  2. Make Sure You're Logged In

    Click "Sign in" in the top-right corner if you're not already logged into GitHub.

Step 2: Launch Codespaces

Now let's create your cloud development environment:

  1. Click the Green "Code" Button

    You'll see it near the top of the repository page.

    The Code button dropdown showing the Codespaces tab

  2. Select the "Codespaces" Tab

    In the dropdown menu, click on the "Codespaces" tab (not "Local").

  3. Click "Create codespace on main"

    This starts building your development environment.

    First Time Setup: The first launch takes 2-3 minutes as it builds the environment. Subsequent launches are much faster (~30 seconds).

Step 3: Welcome to Your Cloud IDE!

Once Codespaces finishes building, you'll see VS Code running in your browser:

Your Codespaces workspace with VS Code in the browser

Understanding the Interface

Area Description
Explorer (Left) File browser showing all project files
Editor (Center) Where you view and edit code files
Terminal (Bottom) Command line interface for running commands
_src Folder Contains all our sample code

Step 4: Navigate to the Sample Code

Let's navigate to the agentquickstart sample using the API Key authentication method:

Using the Terminal

In the terminal at the bottom of Codespaces, type:

Terminal Command
cd _src/useKey/agentquickstart
Why useKey? We're using the API Key authentication method for simplicity. It's perfect for getting started quickly without additional authentication setup.

Verify You're in the Right Place

List the files to confirm:

Terminal Command
ls

You should see:

azure_ai_basic.py
README.md

Step 5: Install Dependencies

The good news? Most dependencies are already installed! But let's make sure we have everything:

Terminal Command
pip install -r ../../requirements.txt

This command:

  • Goes up two directories to find requirements.txt
  • Installs all required Python packages
  • Takes 1-2 minutes to complete
Done! When you see "Successfully installed..." all packages are ready.

Step 6: Create Configuration File

Create a .env file to store Azure OpenAI credentials (we'll get these values in the next module):

Terminal Command
cat > .env << 'EOF'
# Azure OpenAI Configuration
AZURE_OPENAI_ENDPOINT=your-endpoint-here
AZURE_OPENAI_API_KEY=your-api-key-here
AZURE_OPENAI_DEPLOYMENT_NAME=gpt-4o
AZURE_OPENAI_API_VERSION=2024-05-01-preview
EOF
Placeholder Values: Don't worry about the "your-endpoint-here" values. We'll get the real values from Azure Portal in the next module and update this file.

Verify .env File Creation

Check if the file was created:

Terminal Command
ls -la .env

You should see the file listed.

Codespaces Pro Tips

Auto-Save

Your work is automatically saved in the cloud. No need to worry about losing progress!

Stop Codespaces

Codespaces auto-stops after 30 minutes of inactivity. You can also manually stop it from the GitHub Codespaces page to save hours.

Resume Later

You can always resume your Codespace later. All your files and changes will be there!

Port Forwarding

If you run a web server, Codespaces automatically forwards ports so you can access it in your browser.

Setup Checklist

Before moving to the next module, verify:

  • Codespace is running and you see VS Code in your browser
  • You're in the _src/useKey/agentquickstart directory
  • Dependencies are installed (pip install completed successfully)
  • .env file exists with placeholder values
  • Terminal is open and responsive at the bottom of the screen
Excellent! Your cloud development environment is ready. In the next module, we'll get your Azure OpenAI credentials and update the .env file.

Troubleshooting

Codespace Won't Start

  • Check your internet connection
  • Try a different browser
  • Verify your GitHub account has Codespaces enabled

Terminal Not Visible

Press Ctrl + ` (backtick) or go to View Terminal menu.

pip Install Fails

Make sure you're in the correct directory (_src/useKey/agentquickstart) and try running the command again.
