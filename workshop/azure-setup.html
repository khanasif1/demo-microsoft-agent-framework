Module 3: Azure AI Foundry Setup

Create AI project and deploy GPT model

‚è±Ô∏è Duration: 15 minutes

üéØ What You'll Accomplish

In this module, you'll set up Azure AI Foundry, create a project, deploy a GPT model, and configure your application to use it.

üí° What is Azure AI Foundry? It's Microsoft's unified platform for building, deploying, and managing AI applications. It provides access to powerful models like GPT-4, along with tools for evaluation, monitoring, and governance.

‚òÅÔ∏è Step 1: Access Azure AI Foundry

  1. Open Azure AI Foundry

    Go to https://ai.azure.com/ in your web browser.

  2. Sign In

    Sign in with the same Microsoft account you used for Azure CLI (az login).

  3. Accept Terms

    If prompted, accept the terms and conditions to access Azure AI Foundry.

‚úÖ You're in! You should now see the Azure AI Foundry home page.

üèóÔ∏è Step 2: Create an AI Hub

An AI Hub is a workspace that contains all your AI projects and shared resources.

  1. Click "+ New project"

    Look for this button on the home page or in the left navigation.

  2. Select "Create a new hub"

    If you don't already have a hub, you'll be prompted to create one.

  3. Configure Hub Settings

    Fill in the following information:

    Hub name agent-workshop-hub
    Subscription Select your Azure subscription
    Resource group Create new: rg-agent-workshop
    Location Choose nearest region (e.g., East US, West Europe)
  4. Click "Next" and "Create"

    Review your settings and create the hub. This takes 2-3 minutes.

‚ö†Ô∏è Region Selection: Choose a region that supports GPT-4 models. East US, West Europe, and West US are recommended.

üì¶ Step 3: Create an AI Project

  1. Name Your Project

    Once the hub is created, you'll be prompted to create a project:

    • Project name: weather-agent-project
    • Hub: Select the hub you just created
  2. Click "Create"

    Wait for the project to be created (about 1 minute).

‚úÖ Project Created! You now have a dedicated project for your agent.

ü§ñ Step 4: Deploy a GPT Model

Now let's deploy a language model that your agent will use.

Navigate to Model Catalog

  1. Go to "Model catalog"

    In your project, find "Model catalog" in the left sidebar or top navigation.

  2. Find GPT-4o

    Search for or browse to find GPT-4o (or GPT-4o-mini for cost savings).

    üí° Model Options:
    • GPT-4o: Latest and most capable (~$15/1M tokens)
    • GPT-4o-mini: Faster and cheaper (~$0.60/1M tokens) - Recommended for workshop
    • GPT-3.5 Turbo: Budget option (~$1/1M tokens)
  3. Click "Deploy"

    Select the model and click the deploy button.

  4. Configure Deployment
    Deployment name gpt-4o-mini-deployment
    Model version Select latest available
    Deployment type Standard
    Tokens per minute Use default (usually 10K)
  5. Click "Deploy"

    Wait for deployment to complete (1-2 minutes).

‚úÖ Model Deployed! Your GPT model is now ready to use.

üîë Step 5: Get Connection Details

Now we need to collect the information to connect our code to Azure AI.

Get Project Endpoint

  1. Go to Project Settings

    Click on "Settings" or "Overview" in your project navigation.

  2. Find and Copy the Endpoint

    Look for "Project connection string" or "Endpoint". It looks like:

    https://your-project-name.eastus.api.azureml.ms

    Copy this entire URL - you'll need it in the next step.

Get Deployment Name

  1. Go to "Deployments"

    Navigate to the deployments section in your project.

  2. Find Your Deployment

    You should see the deployment you just created (e.g., gpt-4o-mini-deployment).

  3. Copy the Deployment Name

    Note the exact name - it's case-sensitive!

‚öôÔ∏è Step 6: Update .env File

Now let's update the .env file with the real values.

  1. Open the .env File

    In your agentquickstart folder, open the .env file in your text editor.

  2. Replace the Placeholders

    Update the file with your actual values:

    .env file
    # Azure AI Project Configuration
AZURE_AI_PROJECT_ENDPOINT=https://your-actual-project.eastus.api.azureml.ms
AZURE_AI_MODEL_DEPLOYMENT_NAME=gpt-4o-mini-deployment
    ‚ö†Ô∏è Replace these values:
    • AZURE_AI_PROJECT_ENDPOINT: Paste your project endpoint URL
    • AZURE_AI_MODEL_DEPLOYMENT_NAME: Paste your deployment name
  3. Save the File

    Save the .env file. Make sure there are no extra spaces or quotes around the values.

‚úÖ Configuration Complete! Your application can now connect to Azure AI.
