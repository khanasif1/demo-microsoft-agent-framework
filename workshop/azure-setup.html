Module 3: Azure AI Foundry Setup

Create AI project and deploy GPT model

‚è±Ô∏è Duration: 15 minutes
‚ö†Ô∏è Important: Skip this section if you are using instructor provided Azure AI keys and endpoint. Proceed to the Code Walkthrough module.

üéØ What You'll Accomplish

In this module, you'll set up Azure AI Foundry, create a project, deploy a GPT model, and configure your application to use it.

üí° What is Azure AI Foundry? It's Microsoft's unified platform for building, deploying, and managing AI applications. It provides access to powerful models like GPT-4, along with tools for evaluation, monitoring, and governance.

‚òÅÔ∏è Step 1: Access Azure AI Foundry

  1. Open Azure AI Foundry

    Go to https://ai.azure.com/ in your web browser.

  2. Sign In

    Sign in with the same Microsoft account.

  3. Accept Terms

    If prompted, accept the terms and conditions to access Azure AI Foundry.

‚úÖ You're in! You should now see the Azure AI Foundry home page.

üèóÔ∏è Step 2: Create Microsoft Foundry

It contains all your AI projects and shared resources.

  1. Click "+ Create a resource"

    Look for this button on the home page or in the left navigation.

  2. Search "Microsoft Foundry"

    This will take you to marketplace

  3. Click "Create"

    This will take you to wizard to setup your Microsoft Foundry.

  4. Configure Microsoft Foundry

    Fill in the following information:

  5. Click "Create"

    Leave all default and click "Create" to finish setting up your Microsoft Foundry.

‚ö†Ô∏è Region Selection: Choose a region East US recommended. Wait for the Foundry to be created (may take several minutes).

ü§ñ Step 3: Deploy a GPT Model

Now let's deploy a language model that your agent will use.

Navigate to Model Catalog

  1. Navigate to Microsoft Foundry and go to "Model catalog"

    In your project, find "Model catalog" in the left sidebar or top navigation.

  2. Find GPT-4.1

    Search for or browse to find GPT-4.1 (or GPT-4.1-mini for cost savings).

    üí° Model Options:
    • GPT-4.1: Latest and most capable (~$15/1M tokens check updated price)
    • GPT-4.1-mini: Faster and cheaper (~$0.60/1M tokenscheck updated price) - Recommended for workshop
    • GPT-3.5 Turbo: Budget option (~$1/1M tokencheck updated prices)
  3. Click "Use this Model"

    Select the model and click the deploy button.

  4. Configure Deployment
    Deployment name gpt-4.1
    Model version Select latest available
    Deployment type Standard
    Tokens per minute Use default (usually 150K)
  5. Click "Deploy"

    Wait for deployment to complete .

‚úÖ Model Deployed! Your GPT model is now ready to use.

üîë Step 5: Get Connection Details

Now we need to collect the information to connect our code to Azure AI.

Get Model Deployment Details

  1. Stick to Details section of model deployment

    Copy Endpoint URL

  2. Copy Key

    Go to "Keys" and copy the keys.

  3. Copy Deployment name

    Copy the deployment name you used when deploying the model (e.g., gpt-4.1).

‚öôÔ∏è Step 6: Update .env File

Now let's update the .env file with the real values.

  1. Open the .env File

    In your agentquickstart folder, open the .env file in your text editor.

  2. Replace the Placeholders

    Update the file with your actual values:

    .env file
    # Azure AI Project Configuration
AZURE_AI_PROJECT_ENDPOINT=https://{your-endpoint}.openai.azure.com
AZURE_AI_MODEL_DEPLOYMENT_NAME={your-deployment-name}
AZURE_AI_API_KEY={your-api-key}
    ‚ö†Ô∏è Replace these values:
    • AZURE_AI_PROJECT_ENDPOINT: Paste your project name selected
    • AZURE_AI_MODEL_DEPLOYMENT_NAME: Paste your deployment name
    • AZURE_AI_API_KEY: Paste your API key
  3. Save the File

    Save the .env file. Make sure there are no extra spaces or quotes around the values.

‚úÖ Configuration Complete! Your application can now connect to Azure AI.
